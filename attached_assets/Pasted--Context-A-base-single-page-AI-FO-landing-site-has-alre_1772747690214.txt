## 7) API ROUTE HARDENING (app/api/waitlist/route.ts)

**Environment variables:**
- `process.env.AIRTABLE_API_TOKEN`
- `process.env.AIRTABLE_BASE_ID`
- Table name is **hardcoded** as `"Submissions"` — no third env variable needed

**Honeypot:**
- Field name: `companyWebsite`
- If filled → return `200 { ok: true }` silently. Do NOT send to Airtable.

**Rate limiting:**
- 10 requests per IP per hour
- Implement in `lib/rateLimit.ts`
- In-memory rate limiting is acceptable for MVP — prioritize simplicity and build stability
- On limit exceeded → return `429 { ok: false, error: "Too many requests. Please try again later." }`

**Server-side validation:**
- Trim all inputs
- Lowercase email
- Cap all field lengths at 500 characters

**Airtable POST endpoint:**
```
https://api.airtable.com/v0/{AIRTABLE_BASE_ID}/Submissions
```

**Request body:**
```json
{
  "records": [{
    "fields": {
      "Name": "string",
      "Email": "string",
      "Company Size": "string",
      "Accounting System": "string",
      "Submitted At": "ISO timestamp"
    }
  }]
}
```

**Headers:**
```
Authorization: Bearer {AIRTABLE_API_TOKEN}
Content-Type: application/json
```

**Response shape — always use exactly:**
```json
{ "ok": true }
// or
{ "ok": false, "error": "message" }
```

**Error logging rules:**
- On Airtable failure: log only the HTTP status code and request ID if available
- Never log env vars, tokens, or user PII
- Never return stack traces or sensitive details in any response
