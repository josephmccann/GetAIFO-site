## PROJECT OVERVIEW

You are building a production-ready marketing website for **AI.FO** (domain: getaifo.com).

**Goal:** A single-page, high-conversion landing page that matches the AI.FO pitch deck style — dark, minimal, CFO-grade.

Prioritize credibility and clarity: "decision-ready judgment," not "dashboards."

The site should feel like it was designed by a senior fintech product designer, not a generic SaaS template.

**Tone:** authoritative, intelligent, minimal, and spacious.

---

## TECH STACK

- Next.js (App Router) + TypeScript
- Tailwind CSS
- Airtable API for waitlist submissions via a Next.js API route (`/api/waitlist`)
- Airtable token must **NEVER** be exposed client-side
- Must run cleanly with `npm run dev`
- Must deploy cleanly on Replit Deployments
- No additional backend required

## API ROUTE (/api/waitlist/route.ts)

```ts
// POST handler
// Read from environment:
//   process.env.AIRTABLE_API_TOKEN
//   process.env.AIRTABLE_BASE_ID
// Table name is hardcoded as: Submissions

// Server-side validation:
//   - Trim all inputs
//   - Lowercase email
//   - Cap field length at 500 characters
//   - If honeypot field (companyWebsite) is filled → return { ok: true } silently

// Rate limit: 10 requests per IP per hour (use lib/rateLimit.ts)

// POST to:
// https://api.airtable.com/v0/{AIRTABLE_BASE_ID}/Submissions

// Request body:
// {
//   records: [{
//     fields: {
//       Name: string,
//       Email: string,
//       "Company Size": string,
//       "Accounting System": string,
//       "Submitted At": ISO timestamp
//     }
//   }]
// }

// Headers:
//   Authorization: Bearer {AIRTABLE_API_TOKEN}
//   Content-Type: application/json

// Response:
//   Success: { ok: true }
//   Failure: { ok: false, error: "message" }

// Token must NEVER appear in client-side code or responses.
```
